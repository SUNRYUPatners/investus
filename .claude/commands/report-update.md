### 7. 배포 전 팩트체크 (필수 — 생략 불가)

> ⛔ **이 단계 없이 commit/deploy 절대 금지**
> ⛔ **"팩트체크 완료" 선언은 실제로 스크린샷을 다시 Read한 후에만 가능**
> ⛔ **기억·추론으로 대체 절대 금지 — 반드시 원본 스크린샷 파일을 다시 열어서 눈으로 확인**

#### 팩트체크 의무 절차 (순서대로 전부 실행)

**Step 1 — 핵심 수치 목록 추출**
작성한 모든 리포트에서 아래 항목을 표로 정리한다:
| 리포트명 | 수치/가격/% | 출처 스크린샷 파일명 |
|---------|------------|-------------------|
| (예) SPCX RJ PT | $800 | 스크린샷 7:37:36.png |

**Step 2 — 스크린샷 원본 재독 (Read tool 사용 필수)**
위 표의 각 항목마다 해당 스크린샷 파일을 `Read` tool로 **실제로 다시 열어서** 숫자를 직접 눈으로 확인한다.
- 기억에 의존하거나 "아까 봤으니까 맞겠지" 하는 추론 절대 금지
- 스크린샷 글씨가 작아서 불명확하면 불명확하다고 명시하고 가장 보수적인 숫자 사용

**Step 3 — 1:1 대조 체크리스트**
- [ ] 가격·목표주가 → 스크린샷 원문 숫자와 정확히 일치
- [ ] 퍼센트(%) → 방향(+/-)과 수치 모두 일치
- [ ] 날짜 → 원문 날짜와 일치
- [ ] 회사명·종목·사람 이름 → 오탈자 없음
- [ ] 방향성(상승/하락, 매수/매도) → 원문과 동일
- [ ] 없는 사실 창작 여부 → 스크린샷에 없는 내용 0건

**Step 4 — 불일치 발견 시**
즉시 수정 → 해당 SVG·ts 파일 재생성 → 스크린샷 재확인 → 체크리스트 재통과

## 중요 규칙
- 1 리포트 = 1 주제 — 하나의 seed에 여러 주제 섞지 않는다
- **같은 종목이라도 이벤트/주제가 다르면 반드시 별도 리포트로 분리** — 한 seed에 독립적인 여러 이벤트 묶지 않는다 (예: 한국 FSD 구독 개시와 스페인 FSD 승인은 별도 seed)
- 날짜별 타임스탬프는 상대 오프셋으로 계산 (`_now - N*3600_000`)
- 배포 전 TypeScript 오류 0개 확인 필수
- **PNG 내보내기는 오늘 새로 만든 SVG만** — 이전 날짜 이미지 재생성 절대 금지
- **SVG는 `*20260702*.svg`(메인)를 먼저 직접 Read해서 따라 만들고, 애매할 때만 `*20260703*.svg`(서브) 참고 — `*20260704*.svg`는 금지** / 본문에는 항상 상세 글 + 투자시사점 (스킬: `.cursor/skills/investus-report-update/`)

## 종토방(wallPosts.ts) 글 작성 규칙
- 닉네임은 `익명_XXXX` 형식으로 4자리 숫자 무작위하게 배정
- 말투는 구어체 자연스럽게, 분석+개인 경험+감상 섞기
- 같은 날 같은 ticker 게시글이라도 논점/각도가 서로 달라야 한다
- holdingLabel은 "관심종목", "XX주 보유" 등 다양하게 배분

## 애널리스트(analystPosts.ts) 글 작성 규칙 — 절대 금지 패턴
- **금지**: 같은 날 같은 alias 두 번 이상 사용 금지 (예: "여의도 매 #17"을 같은 날 반복 불가)
- **금지**: 모든 글을 "~정리임", "~세트임"으로 시작하거나 끝내는 패턴 반복 금지
- **금지**: 모든 글이 "X종 세트임", "Y가지 정리" 식의 같은 구조로 시작하는 것 금지
- **다양한 오프닝 스타일** (날짜별로 섞어서 사용):
  - 채널 체크 뷰: "~쪽 채널 체크해봤는데", "~딜 플로우 확인했는데"
  - 숫자/데이터 포인트: "~보고 한 가지 숫자가 눈에 들어왔어"
  - 심층 분석: "~발언 다시 뜯어봤는데"
  - 검증: "~결과랑 딱 맞아 떨어져서"
  - 현장감: "~사진 봤는데, 속도가 예상보다"
  - 단도직입: "~가 오늘 핵심이야. 이유는..."
- 글마다 다른 관점(강세/중립/분석/의문) 배분 — 전부 강세로 쓰지 않는다
- alias는 ALIAS_POOL 전체에서 고르게 배분 (특정 alias 반복 지양)
