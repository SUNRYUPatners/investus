# Investus 리포트 업데이트 스킬

리포트 날짜 세트를 추가할 때 **항상** 이 스킬 + `.claude/commands/report-update.md` 전체 순서를 따른다.

## SVG 이미지 — 7월2일 = 메인 기준, 7월3일 = 서브 (2026-07-16 업데이트: 레이아웃 다양화 필수)

**7/4 스타일(배경 그리드 라인 + "핵심사실/투자시사점" 2단 카드 박스)은 더 이상 쓰지 않는다.** 7/15 리포트를 7/4 스타일로 만들었다가 반려됨 — 이유: 지나치게 표/인포그래픽 느낌이라 딱딱함.

**⚠️ 하나의 획일화된 템플릿만 반복 사용 금지 (2026-07-16 반려 사유).** 7/16 리포트를 만들 때 16개 전부를 "이모지 히어로 + KPI 카드 2개 + 의미 박스" 하나의 틀로만 찍어냈다가 반려됨 — 이유: "7/2 안에도 다양한 형식이 있는데 왜 다 똑같은 형식이냐". **7/2 하루치 안에서도 실제로는 최소 6~9가지 서로 다른 레이아웃이 쓰였다.** 매일 개별 리포트 16개 정도를 만든다면, 그 안에서 최소 4~6가지 이상의 레이아웃을 섞어 써야 한다.

**우선순위: 7/2가 메인 기준, 7/3은 서브(보완) 참고용.**
- 항상 **먼저 `*20260702*.svg` 전체 파일을 다 Read로 훑어서** 오늘 쓸 수 있는 레이아웃 종류를 파악한다 (1~2개만 보고 그걸로 전체를 찍어내지 말 것).
- 7/2 파일 중 오늘 소재와 비슷한 게 없을 때만 `*20260703*.svg`를 보조로 참고한다.

**필수 레퍼런스 — 7/2 안에 있는 대표 레이아웃 6종 (매번 새 리포트 세트를 만들기 전에 전부 Read로 열어서 구조 파악):**
1. **숫자 히어로 + KPI 카드 2개 + 의미박스** — `meta-cloud-ai-20260702.svg` (가장 기본형, 남용 금지 — 하루 세트의 25~30% 이하로만 사용)
2. **숫자 히어로 + 3-스탯 카드 나열 + 상세박스** — `tsla-optimus-fremont-20260702.svg`, `tsla-supercharger-q2-20260702.svg` (지표 3개 이상 있는 소재)
3. **체크마크/문구 히어로 + 3-피처 카드 + 인용박스** — `tsla-fsd-ae3-20260702.svg` (기능 출시·마일스톤 달성 소재)
4. **배지 + 짧은 문구 히어로 + 큰 인용박스 + 스탯칩** — `elon-ai-power-20260702.svg` (발언/코멘트 중심 소재)
5. **Before→After 비교 박스(화살표)** — `elon-ai-power-20260702.svg`의 비교 섹션 참고해 응용 (수치 대비 소재: A vs B, 이전 vs 현재)
6. **BREAKING 배지 + 큰 히어로 + 국가/카테고리 그리드** — `tsla-europe-june-20260702.svg` (복수 항목 랭킹·비교 소재)
7. (선택) **큰 range/D-DAY 히어로 + 컨텍스트 박스** — `tsla-q2-delivery-day-20260702.svg` (예정된 이벤트/발표 대기 소재)
8. (선택) **제품/기기 목업 일러스트 카드** — `spcx-ai-device-20260702.svg` (신제품·프로토타입 노출 소재, 자주 쓸 필요 없음)

`*20260704*.svg` 파일은 참고하지 말 것.

**공통 뼈대 (레이아웃과 무관하게 항상 유지할 것):**
1. 배경: 단순 그라디언트 + 얇은 테두리 rect. **배경 그리드 라인(격자) 넣지 않음.**
2. 상단 5px 컬러 바
3. 헤더: 좌측 티커 배지(pill) + 중앙 "INVESTUS DAILY REPORT" 라벨 + 우측 날짜 배지. (BREAKING류는 대신 상단 중앙에 배지 하나)
4. 제목 1줄 (28~30px, 흰색, 굵게), 얇은 구분선
5. 본문은 위 6~8가지 레이아웃 중 **소재 성격에 가장 잘 맞는 것을 골라서** 구성 (아래 "소재별 레이아웃 매칭" 참고)
6. **"투자 시사점"이라는 라벨을 이미지에 박지 않는다** — 투자 판단에 도움되는 맥락은 박스 안에 자연스러운 문장으로 녹여서 넣는다.
7. 하단: 요약 caption 한 줄(21~22px, `#374151`, y=960 근처로 하단 컬러바에 가깝게 배치해 빈공간 최소화) + 하단 컬러 바(20px, opacity .4, y=1060) + 트레이드마크 텍스트(11px, letter-spacing, `investus.kr SRP 최고투자책임자 발행` / EN `SRP Chief Investment Officer`)

**소재별 레이아웃 매칭 가이드:**
- 지표 1개가 핵심 → 레이아웃 1 (숫자 히어로 + KPI 2개)
- 지표 3개 이상(실적 발표, 생산 스케일업 등) → 레이아웃 2 (3-스탯 카드)
- 인물 발언/코멘트가 핵심(버핏, 젠슨 황 등) → 레이아웃 4 (인용박스+칩) 또는 배지+문구 히어로
- 기능 출시·마일스톤 완주 → 레이아웃 3 (체크마크+피처카드)
- A vs B 대비가 핵심(가격 인하폭 비교, 설문 응답 비교, IPO가 대비 현재가 등) → 레이아웃 5 (Before→After 화살표)
- 순위/1위 발표·전월 대비 급등 등 임팩트 큰 뉴스 → 레이아웃 6 (BREAKING 배지 + 큰 히어로)

**작업 순서:**
1. 오늘 다룰 소재 16개(또는 해당 개수)를 확인하고, 각 소재를 위 매칭 가이드로 레이아웃 1~6에 분배한다 (하나의 레이아웃이 전체의 40%를 넘지 않게)
2. 레이아웃별로 해당하는 7/2 파일을 Read로 열어 정확한 구조·좌표를 확인한다
3. 그 구조를 그대로 이번 소재 데이터로 채워서 생성 — "핵심사실/투자시사점" 2단 카드나 배경 그리드는 넣지 않음
4. 완성 후 sharp로 PNG 렌더링해서 실제로 눈으로 확인 (잘림/오버플로/이모지 깨짐/레이아웃 겹침 체크)
5. **이모지 주의**: `scripts/export-report-pngs.js`의 `stripEmoji()`가 `\u{1F000}-\u{1FFFF}`, `\u{2600}-\u{27BF}`, `\u{FE00}-\u{FE0F}` 범위만 제거한다. 이 범위 밖 이모지(예: ⏳ U+23F3)는 stripped되지 않고 깨진 fallback 글리프(검은 박스)로 렌더링되니 반드시 위 범위 안의 이모지만 사용할 것.

## 본문(body) 원칙 — 변경 없음

1. **본문(`lib/reports.ts` body/bodyEn)에는 상세 설명·배경·출처·투자시사점을 충분히 채운다.**
2. 모든 개별 리포트에 `■ 투자시사점` (EN: `■ Investment Implications`) 필수 — SVG 안에는 라벨 없이 자연스럽게 녹여도, 본문에는 항상 명시적 섹션으로 있어야 함.

## 실행 체크리스트 (빠지면 미완성)

1. `01.investus 리포트/` 원본 스크린샷 Read로 팩트 추출
2. **7/2 SVG(메인)을 먼저 Read로 열어보고**, 애매하면 7/3(서브)도 참고해 구조를 따라 SVG KO+EN 작성 (7/4 스타일 금지)
3. `lib/reports.ts` seed + tickers — **본문 충실 + 투자시사점**
4. `lib/wallPosts.ts` 글/댓글 + `LATEST_UPDATE`
5. `lib/analystPosts.ts` 개별 리포트마다 1개 이상
6. `scripts/export-report-pngs.js` PAIRS (오늘 date만)
7. `node scripts/export-report-pngs.js` (**`--all` 금지**)
8. 팩트체크: 스크린샷 재 Read 후 수치 1:1 대조
9. PNG 렌더링해서 실제 눈으로 레이아웃 확인 (잘림/빈공간 체크)
10. commit + `bash scripts/deploy.sh` (필요 시 `--notify`)
