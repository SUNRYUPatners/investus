# Investus 리포트 업데이트 스킬

리포트 날짜 세트를 추가할 때 **항상** 이 스킬 + `.claude/commands/report-update.md` 전체 순서를 따른다.

## SVG 이미지 — 7월2일 = 메인 기준, 7월3일 = 서브 (2026-07-15 업데이트, 7/4 스타일 금지)

**7/4 스타일(배경 그리드 라인 + "핵심사실/투자시사점" 2단 카드 박스)은 더 이상 쓰지 않는다.** 7/15 리포트를 7/4 스타일로 만들었다가 반려됨 — 이유: 지나치게 표/인포그래픽 느낌이라 딱딱함.

**우선순위: 7/2가 메인 기준, 7/3은 서브(보완) 참고용.**
- 항상 **먼저 `*20260702*.svg`를 기준으로** 구조를 잡는다.
- 7/2 파일 중 오늘 소재와 비슷한 게 없을 때만 `*20260703*.svg`를 보조로 참고한다.

**필수 레퍼런스 (매번 새 리포트 만들기 전에 최소 1~2개씩 Read로 열어볼 것):**
- **메인**: `public/charts/*20260702*.svg` (예: `tsla-q2-delivery-day`, `spcx-wedbush-150`, `meta-cloud-ai`, `tsla-semi-europe`, `tsla-optimus-fremont`)
- **서브**: `public/charts/*20260703*.svg` (예: `msft-frontier`, `tsla-optimus-v3-hand`)

`*20260704*.svg` 파일은 참고하지 말 것.

**7/2·7/3 스타일의 실제 구조 (이 순서·형태를 그대로 따른다):**
1. 배경: 단순 그라디언트 + 얇은 테두리 rect. **배경 그리드 라인(격자) 넣지 않음.**
2. 상단 5px 컬러 바
3. 헤더: 좌측 티커 배지(pill) + 중앙 "INVESTUS DAILY REPORT" 라벨 + 우측 날짜 배지. (BREAKING류는 대신 상단 중앙에 배지 하나, 예: `tsla-q2-delivery-day-20260702.svg` 참고)
4. 제목 1줄 (28~30px, 흰색, 굵게), 얇은 구분선
5. **히어로 섹션**: 이모지(60~70px) + 큰 숫자/텍스트를 **같은 자리에 두 번 그려서** 은은한 글로우 효과(하나는 opacity .12로 배경, 하나는 opacity 1 전경) — 100~130px, 티커 강조색. 그 아래 22~26px 회색 서브텍스트 한 줄. 얇은 구분선.
6. **KPI 카드 2개** 나란히 (각 440~460px 폭, 160~220px 높이): 라벨(22~24px 회색) + 큰 값 1개(36~56px, 굵게) + 서브텍스트(20~22px 회색). "핵심사실 불릿 리스트"가 아니라 **핵심 숫자/키워드 하나를 크게** 보여주는 카드.
7. **의미/코멘트 박스** 1개 (전체 폭 960px, 170~220px 높이): 상단에 작은 라벨("의미" / "OOO 코멘트" 등, 22px 회색 letter-spacing) + 이모지 붙은 팩트 라인 2~3개(26px) + 출처 한 줄(22px 회색) + 필요시 인용구 한 줄(18~20px 더 어두운 회색). **"투자 시사점"이라는 라벨을 이미지에 박지 않는다** — 투자 판단에 도움되는 맥락은 이 박스 안에 자연스러운 문장으로 녹여서 넣는다.
8. 하단: 요약 caption 한 줄(22px, `#374151`) + 하단 컬러 바(20px, opacity .4) + 트레이드마크 텍스트(11px, letter-spacing, `investus.kr SRP 최고투자책임자 발행`)

**작업 순서:**
1. 오늘 다룰 소재들을 확인
2. **먼저 7/2 폴더**에서 소재 성격이 비슷한 SVG 1~2개를 Read로 열어 구조 확인 (7/2에 적당한 게 없으면 7/3도 같이 확인)
3. 그 구조(헤더 → 제목 → 히어로(글로우 이펙트) → KPI 카드 2개 → 의미 박스 → 캡션+트레이드마크)를 그대로 이번 소재 데이터로 채워서 생성 — "핵심사실/투자시사점" 2단 카드나 배경 그리드는 넣지 않음
4. 완성 후 sharp로 PNG 렌더링해서 실제로 눈으로 확인 (잘림/오버플로 체크)

## 본문(body) 원칙 — 변경 없음

1. **본문(`lib/reports.ts` body/bodyEn)에는 상세 설명·배경·출처·투자시사점을 충분히 채운다.**
2. 모든 개별 리포트에 `■ 투자시사점` (EN: `■ Investment Implications`) 필수 — SVG 안에는 라벨 없이 자연스럽게 녹여도, 본문에는 항상 명시적 섹션으로 있어야 함.

## 한장요약 SVG

- **메인 레퍼런스**: `public/charts/summary-20260702.svg` (색상 리스트 행, 좌측 컬러 바 + 이모지 + 제목 + 서브텍스트) — 이 형태를 기본으로 사용
- **서브 레퍼런스**: `summary-20260703.svg` (브레이킹 히어로 + 카드 그리드) — 7/2 형태로 담기 애매한 경우만 보조로 참고
- `summary-20260704.svg`는 참고하지 말 것

## 실행 체크리스트 (빠지면 미완성)

1. `01.investus 리포트/` 원본 스크린샷 Read로 팩트 추출
2. **7/2 SVG(메인)을 먼저 Read로 열어보고**, 애매하면 7/3(서브)도 참고해 구조를 따라 SVG KO+EN 작성 (7/4 스타일 금지)
3. `lib/reports.ts` seed + tickers — **본문 충실 + 투자시사점**
4. `lib/wallPosts.ts` 글/댓글 + `LATEST_UPDATE`
5. `lib/analystPosts.ts` 개별 리포트마다 1개 이상
6. `scripts/export-report-pngs.js` PAIRS (오늘 date만)
7. `node scripts/export-report-pngs.js` (**`--all` 금지**)
8. 팩트체크: 스크린샷 재 Read 후 수치 1:1 대조
9. PNG 렌더링해서 실제 눈으로 레이아웃 확인 (잘림/빈공간 체크)
10. commit + `bash scripts/deploy.sh` (필요 시 `--notify`)

